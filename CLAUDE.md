## Project Overview

**Automancy** is a TypeScript converter that transforms natural language D&D 5e homebrew ability descriptions into fully automated Foundry VTT items. Output is compatible with MidiQOL, DAE, Chris Premades, and Gambit's Premades modules.

**Current status**: Phase 2 Professional system is fully implemented. The converter handles weapon attacks, spell attacks, saves, conditions (prone, grapple, restrained, dazed), ongoing damage, recharge mechanics, reactions, and creature stat blocks.

## Build & Run Commands

```bash
npm run build          # Compile TypeScript (tsc)
npm run dev            # Watch mode - recompile on changes
npm run clean          # Remove dist/
npm start              # Run demo (dist/index.js)
npm run cli demo       # CLI demo with 4 test cases
npm run cli convert "ability text" [name] [--output file.json]
npm run cli batch input.txt [--output output.json]
npm run cli creature statblock.txt [--output creature.json]
npm run cli capabilities   # Show supported patterns
npm run lint           # ESLint
npm test               # Jest (no test files exist yet)
```

Always `npm run build` before running - the CLI runs from `dist/`.

## Architecture

```
Text Input → TextAnalyzer (parsing) → Phase2Converter (orchestration) → Foundry JSON Output
```

### Pipeline Flow

1. **TextAnalyzer** (`src/parser/text-analyzer.ts`) parses raw text into a `ParsedAbility` struct using **PatternMatcher** (`src/parser/pattern-matcher.ts`) which has 20+ regex patterns for D&D text formats.

2. **Phase2Converter** (`src/phase2-converter.ts`) orchestrates 8 automation subsystems that each enhance the base conversion:
   - `ConditionEngine` - Prone, grapple, restrained, dazed, specter summoning
   - `ComplexAbilityParser` - Multi-step requirements, linked effects
   - `ProfessionalFlags` - MidiQOL, DAE, chris-premades, gambit flags
   - `MacroTemplateSystem` - OnUse macro generation for complex abilities
   - `RechargeAutomation` - Recharge 4-6, 5-6 mechanics
   - `ReactionTracking` - Reaction consumption and triggers
   - `OngoingEffectsTracking` - Turn-based persistent effects
   - `MovementAttackIntegration` - Movement requirement validation

3. **RuleEngine** (`src/automation/rule-engine.ts`) generates the final Foundry item JSON with Activities (D&D 5e 4.x compatible).

### Entry Points

- `AutomancyConverter` (`src/index.ts`) - Public API. Delegates to Phase2Converter.
- `AutomancyCLI` (`src/cli.ts`) - CLI interface with convert/batch/creature/demo commands.
- `CreatureConverter` (`src/creature-converter.ts`) - Full stat block → Foundry actor conversion.

### Key Types

All in `src/types/index.ts`:
- `ParsedAbility` - Structured representation of a parsed ability
- `FoundryItemData` - Complete Foundry item schema
- `ActiveEffectData` / `ChangeData` - Effect structures
- `AbilityType` enum - weapon_attack, spell_attack, save_ability, healing, utility, passive, reaction
- `AutomationComplexity` enum - SIMPLE(1), MODERATE(2), COMPLEX(3), REACTION(4)

Creature types in `src/types/creature-types.ts`.

## Complexity Assessment

The system rates abilities 1-4 and chooses automation strategy accordingly:
- **Level 1**: Flag-based only (simple attacks, basic bonuses)
- **Level 2**: Simple macros + conditional effects
- **Level 3**: Advanced macros, multi-step workflows
- **Level 4**: Reaction-based with real-time triggers

## Module Compatibility Patterns

Generated output must follow these module conventions:
- **MidiQOL**: Flags at `flags.midi-qol.*` (advantage, saves, damage workflows). Timing hooks: preAttackRoll, postDamageRoll, etc.
- **DAE**: Enhanced Active Effects with `flags.dae.*` (stackable, specialDuration, macroRepeat).
- **Chris Premades**: `flags.chris-premades.*` with condition/escapeDC patterns. "Medkit" integration.
- **Gambit's Premades**: Reaction triggers via `flags.gambits-premades.*`.
- **Convenient Effects**: Custom status effects via `flags.convenient-effects.*`.

## Notes

- TypeScript strict mode is enabled. All source is in `src/`, output in `dist/`.

- No formal Jest test suite exists yet - only manual testing via demos and test creature files.
